Overview

This tool tells you how many parts to inspect from a lot so that your findings are statistically meaningful. Instead of guessing or inspecting everything, you pull a calculated sample — and the result either clears the lot or condemns it.

Core principle: You are not trying to find every bad part. You are trying to get statistical evidence about whether the lot as a whole is acceptable or not.
Step-by-Step Instructions
  1. 1
    Enter your lot size

    Type in the total number of parts in the batch. Default is 700 pcs. This can be any batch size.

  2. 2
    Choose your confidence level

    90% — Quick directional check. Low-stakes parts.
    95% — Industry standard. Use this for most situations.
    99% — Safety-critical parts where a bad part could cause injury or major failure.

  3. 3
    Set the defect rate threshold

    Ask yourself: "At what percentage of bad parts does this become a real problem?" If even 1-in-20 bad (5%) is unacceptable, set it to 5%. The lower the threshold, the larger the required sample.

  4. 4
    Pull your sample — randomly

    Do not cherry-pick good-looking parts. Do not pull only from one spot in the bin or pallet. Select parts from different positions throughout the lot.

  5. 5
    Inspect each part against your defect criteria

    Use your standard inspection criteria. Count and record how many parts are nonconforming.

  6. 6
    Apply the decision rule

    The "Accept if ≤" number is your threshold — currently always 0. Zero defects found = PASS. One or more = FAIL. See the Pass/Fail tab for actions.

Probability that your test detects a problem if one truly exists.
Defect Rate (p)
Minimum bad-part % you want to be able to detect.
Accept Number (c)
Max defects allowed in sample to pass. This tool uses c = 0.
Reject Number
Any result above the accept number — reject the lot.
The Decision Rule
PASS: You inspect your sample and find zero (0) defective parts. Release the lot. You have statistically demonstrated — at your chosen confidence level — that the defect rate is likely below your threshold.
FAIL: You find one (1) or more defective parts in your sample. Reject the lot. The sample provides statistical evidence that the lot defect rate exceeds your threshold.

This tool uses an acceptance number of c=0 — the most conservative and common approach for internal QC. Any defect in the sample is a rejection trigger.

What To Do When It Fails
  1. A
    100% Sort

    Inspect every single part in the lot. Pull all nonconforming parts. Use when parts are recoverable and the sort cost is justified.

  2. B
    Return to Supplier

    Send the lot back with your sample findings as evidence. Document the defect count, type, and sample size for your records and any supplier corrective action request (SCAR).

  3. C
    Scrap the Lot

    If sorting cost exceeds part value, or the defect is safety-critical with zero tolerance, condemn and scrap the entire lot.

  4. D
    Escalate

    If 3+ defects are found, this may indicate a systemic problem. Escalate to engineering or supplier quality for formal corrective action.

Important Caveats
A PASS does not mean the lot is perfect. It means you have statistical confidence that the defect rate is below your threshold. There may still be some bad parts — you have evidence the rate is acceptably low.
Random sampling is critical. If you bias your sample toward parts that look good, your results are meaningless. The math only works if the sample is truly random.
One defect = reject. There is no "it was only one." The accept number is zero. A single nonconforming part in the sample is statistically sufficient to reject the lot.
Why Not Just Use a Fixed Percentage?

A common approach is to inspect "10% of the lot." The problem: that rule has no statistical basis. A 10% sample of 100 parts gives you very different confidence than a 10% sample of 10,000 parts. The correct approach ties sample size to a specific confidence level and a defect rate you care about detecting.

The Hypergeometric Distribution

When sampling without replacement from a finite lot, the correct probability model is the Hypergeometric distribution — not the Binomial (which assumes an infinite population).

```
Variables
N
Total lot size
D
Assumed defectives in lot = N × p
n
Sample size (what we solve for)
p
Defect rate threshold (e.g. 0.05)
Core Probability Formula

Probability of drawing n parts and finding zero defects, given D defective parts exist:

// P(X = 0) — probability of missing ALL defects in sample C(D, 0) × C(N−D, n) ``` P(X=0) = ───────────────────── C(N, n) // Simplified to a running product (avoids huge factorials): P(X=0) = ∏ (N−D−i) / (N−i) for i = 0, 1, 2 … n−1
```
What We Actually Solve For

We want the probability of detecting at least one defect to meet or exceed the confidence level:

P(detect ≥ 1) = 1 − P(X=0) ≥ confidence ``` // Find the smallest n where this holds true: 1 − P(X=0) ≥ confidence level // Example: N=700, p=5%, conf=95% // → D = 700 × 0.05 = 35 defective parts assumed // → Solve for n: answer is 57 parts
```
Why not the Binomial? The Binomial assumes sampling with replacement — effectively an infinite population. For finite lots, the Hypergeometric is exact. The difference matters most when your sample is a significant fraction of the lot, which it often is in QC.
```
Worked Example
// Given: N = 700 // lot size p = 0.05 // 5% defect rate threshold D = 35 // assumed defectives (700 × 0.05) conf = 0.95 // 95% confidence required // For n = 57, compute the running product: P(X=0) = (665/700) × (664/699) × … [57 terms] = 0.049 // ~4.9% chance of missing all defects P(detect ≥1) = 1 − 0.049 = 0.951 // 95.1% ✓ meets 95% threshold // For n = 56: P(detect ≥1) = 94.8% ✗ just under threshold // Therefore: minimum valid sample = 57 parts

Key insight: Once lot size gets large (500+), sample size grows very slowly. Doubling the lot from 700 → 1400 barely changes the required sample. You don't need to inspect a fixed percentage — the math takes care of it.
